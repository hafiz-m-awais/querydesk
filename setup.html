Set up QueryDesk for your course

Your data stays in your own Google account — no shared server, no cost.
Complete steps 1–4 once, then use the link generator to share with your students.

~10 minutes total
1
Create a Google Sheet
This becomes your course database
  1. Go to sheets.google.com → click Blank spreadsheet
  2. Rename it anything — e.g. QueryDesk – Spring 2026
  3. Leave the first tab as-is — the script will set everything up on first use
Keep this tab open — the next step opens directly from it.
2
Add the 6 script files
Copy from the repo into the Apps Script editor
  1. In your Sheet: Extensions → Apps Script
  2. Delete the default Code.gs (right-click → Delete)
  3. For each row in the table below: click + (Files) → Script, name it, paste the contents
File name to createPaste the contents of
Configgas/Config.gs
GetHandlersgas/GetHandlers.gs
PostHandlersgas/PostHandlers.gs
SheetUtilsgas/SheetUtils.gs
DriveUtilsgas/DriveUtils.gs
EmailUtilsgas/EmailUtils.gs

Press Ctrl+S after each file. Name the project QueryDesk.

All 6 files share the same scope in Apps Script — order doesn't matter.
3
Set your password & email
Stored securely — never in code or GitHub
  1. In Apps Script: click the gear icon ⚙ (Project Settings) in the left sidebar
  2. Scroll to Script Properties → click Add script property
  3. Add both properties from the table below
  4. Click Save script properties
Property nameValue
ADMIN_PASSWORD A strong password you'll use to log into your admin panel
INSTRUCTOR_EMAIL Your email address — you'll receive a notification for each new query
Security: Script Properties are stored inside Google's secure vault and are never visible in source code, version history, or GitHub.
4
Deploy as Web App
Creates your personal backend API URL
  1. Click Deploy (top right of Apps Script) → New deployment
  2. Click the gear ⚙ next to "Select type" → choose Web app
  3. Set:
    • Execute as: Me
    • Who has access: Anyone
  4. Click Deploy → authorize when prompted (click through all permission screens)
  5. Copy the Web App URL — it ends with /exec
Tip: After editing any script file later, you must create a new deployment (Deploy → New deployment again) — not "Manage deployments → Redeploy". A new deployment creates a new URL, so update your links accordingly.
5
Get your personal links
Paste your Web App URL — links ready instantly

Configure your course settings
Set course name, sections, email domain, announcement banner, and more

Once you have your admin panel open:

  1. Log in with the ADMIN_PASSWORD you set in step 3
  2. Click ⚙ Settings
  3. Under Course Settings, fill in:
FieldWhat it does
Course NameShown on the student form header
Term / Semestere.g. Fall 2026
Session Label"Lab", "Lecture", "Week", etc.
Session CountTotal number of sessions (for the dropdown)
SectionsComma-separated: CS-4A, CS-4B
Instructor NameShown in email notifications
Email DomainStudent email suffix — leave blank to accept any
University NameShown in the top-bar — leave blank to hide
Roll/ID FormatRegex for roll validation — leave blank to accept any
Submissions OpenToggle to open or close submissions
AnnouncementInfo banner shown on the student form
All settings are saved to your Google Sheet — no code editing required.
